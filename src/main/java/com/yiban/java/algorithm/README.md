## 负责度大欧O
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(2^n) < O(n!) < O(n^n)
最后两个不常见，可以忽略
- O(1) : 常数时间复杂度(也叫常数阶)，操作时间与数据量无关。比如一个班有50个学生，我直接喊班长，一次就能找到，当这个班增加到100人，我喊班长，还是一次就能找到。典型的就是数组的访问。
- O(log n) : 对数时间复杂度(对数阶)，操作时间与数据量的对数成正比(这里的对数就是log2(n))。典型的例子就是二分查找。
- O(n) : 线性时间复杂度(线性阶)，操作时间与数据量成正比。典型的例子就是一次for循环
- O(n log n) : 线性对数时间复杂度(线性对数阶)，操作时间与数据量乘以其对数成正比。典型的例子就是快速排序(外层for循环+内层二分查找)。
```aiignore
伪代码：
int cn = 1;
for (int i = 1; i <= n; i++) {//外层执行n次
    while(cn <= n) {
        cn = cn * 2; //内层执行log2(n)次
    }
}
```
- O(n^2) : 平方时间复杂度(平方阶)，操作时间与数据量的平方成正比。典型的例子就是冒泡排序(两层嵌套for循环)。
- O(2^n) : 指数时间复杂度(指数阶)，操作时间与2的n次方成正比。
- O(n!) : 阶乘时间复杂度，操作时间与n的阶乘成正比。
- O(n^n) : n的n次方时间复杂度，操作时间与n的n次方成正比。
